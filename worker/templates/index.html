      📚 AI Research Assistant Guide

      Open this anytime from the ? button next to the theme toggle.

      Layout
      • Left sidebar: Saved chat sessions. Click a row to reopen that conversation.
      • New chat: Saves the current workspace if it has any content (messages, topic search results, notes, or uploads), then clears all fields so you start clean.
      • Main column: One workspace — discovery, optional context/reference PDFs, message thread, and question box.
      Discover papers
      • Enter a research topic and run search (currently wired to arXiv discovery on the server).
      • Adjust sources (multi-select), categories, sort, citation threshold, and max results.
      • Results appear below; select papers and use Ingest selected papers (sign-in required) to generate per-paper analysis and optional final report sections further down the page.
      Context & reference PDFs
      • Analysis & reports: Generated by the workflow (discover → ingest → sections below), not uploaded by you.
      • Context / pasted text: Optional notes or excerpts for the model.
      • Reference PDFs: The only optional uploads; sign-in required. Use Download next to a file to save your copy from the server.
      Questions & AI behavior

      Use Your question / Send for the research assistant. The model uses discovery results, optional context and reference PDF names, and any on-screen analysis or final-report text the app has produced. Replies appear in the thread above the input. The floating chat button is a separate helper for search tips, prompts, and n8n/Groq usage.

      Account & data
      • Sign in is required for PDF upload, ingest, downloads, and server-side history where enabled.
      • Chat list and session fields are stored in this browser (localStorage) unless the server also records history.
      • Use Sign out from the top bar when finished on a shared computer.
